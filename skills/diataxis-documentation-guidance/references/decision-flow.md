# Diataxis Decision Flow

Use this flow to assign one primary mode.

## Step 1: Determine user intent axis

- **Study intent**: user is learning concepts or acquiring understanding.
- **Work intent**: user is trying to complete a task now.

## Step 2: Determine cognition axis

- **Action-heavy**: user needs procedural steps.
- **Cognition-heavy**: user needs conceptual framing, definitions, or background.

## Step 3: Map to Diataxis mode

- **Tutorial**: study + action. Goal is guided learning by doing.
- **How-to guide**: work + action. Goal is task completion.
- **Reference**: work + cognition. Goal is quick lookup of facts.
- **Explanation**: study + cognition. Goal is understanding why/how things work.

## Step 4: Resolve hybrid documents

If content spans modes, select one primary mode by dominant user intent (study/work + action/cognition), then split the rest into linked companion pages.

Use this response structure for classification-only output:

1. Category
2. Why
3. Split recommendations
4. Next step
